# 시나리오 제목 편집 키트 (SRW J)

## 기존 추출본이 깨졌던 이유 (구조 분석 결과)

화 n(0~67)의 제목은 **IMG(3462+n) + SCR(3531+n) 페어**입니다.
IMG는 단순한 8x8 타일 풀이고, 화별 SCR 타일맵이 화면 배치를 전적으로 결정합니다.

- 게임은 IMG를 VRAM(한 행 = 32타일)에 행 단위로 적재합니다.
  저장 타일 s ↔ VRAM 번호 = (s ÷ tw)×32 + (s mod tw)
- SCR(30x20)은 화면 타일행 7,8(1행 = 第N話), 11,12(2행 = 부제),
  59화만 13,14(3행)까지 사용합니다.
- 긴 제목(th=3 자산)은 부제의 초과분과 第N話를 **3번째 타일행에 흩어 저장**하고
  SCR이 임의 순서로 끼워 맞춥니다. 일부 화는 좌우반전 플립으로 타일을 재활용합니다.
- 그래서 IMG만 행우선으로 펼치면 "반 잘림 / 순서 뒤섞임"이 생겼던 것입니다.

이번 추출본은 SCR을 통해 **게임 화면에 보이는 그대로** 복원한 것입니다.

## 파일 규격 (e00_img03462.png ~ e67_img03529.png)

- RGBA 240x64. 캔버스 y=0 이 화면 타일행 7 (화면 y=56)
  - y 0~15   : 1행 (제N화)
  - y 32~47  : 2행 (부제)
  - y 48~63  : 3행 (필요 시. 원본은 59화만 사용)
  - y 16~31  : 줄 사이 간격(비워 두세요)
- 투명(알파 0) = 배경. 잉크 = 회색조: 거의 검정(17) = 외곽선, 흰색(255) = 본문.
  중간 회색은 안티앨리어스 단계로 그대로 사용 가능 (밝기 ÷ 17 → 색 인덱스 1~15 환원)
- **텍스트 줄은 반드시 2타일행(16px) 안에**: 1행은 y0~15, 2행은 y32~47을 벗어나면 안 됨
- 가로 위치는 8px 격자에 안 맞아도 됩니다 (삽입기가 셀 단위로 자동 배치)

## 용량 한도

화별 "잉크가 있는 8x8 셀 수 ≤ 타일수-1" (추출 시 콘솔에 한도 출력됨, 26~74개).
초과 시 삽입기가 화번호와 함께 거부합니다. `--grow` 옵션으로 3타일행(95타일)까지
확장 시도 가능(게임에서 검증된 한도). 한글은 일본어보다 획이 단순해 보통 여유 있음.

## 사용법

```
# 추출 (이미 완료된 결과물이 동봉됨)
python scn_title_extract.py rom.gba 시나리오제목_편집용 --sheet

# 편집 후 삽입 (폴더 통째 또는 개별 파일)
python scn_title_insert.py rom_in.gba rom_out.gba 시나리오제목_편집용
python scn_title_insert.py rom_in.gba rom_out.gba e00_img03462.png e01_img03463.png
```

삽입기는 IMG 타일과 SCR 타일맵을 함께 재생성하므로 원본의 열 배치 제약이 없습니다.
중복/플립 타일 자동 공유, ECD 재압축, 슬롯 초과 검사, 기록 후 라운드트립 검증까지 자동.

## 알아둘 점

- 59화 PNG 좌하단(y48 부근)의 작은 픽셀 조각은 원본 데이터의 잔재입니다. 지워도 됩니다.
- 0번 타일은 화면 빈칸 채움용으로 항상 공백이어야 하는데(1화 점선 버그의 원인),
  삽입기가 구조적으로 보장하므로 신경 쓸 필요 없습니다.
- PNG 색은 편집용 회색조입니다. 실제 게임 색(흰 본문/어두운 외곽선 + 불 배경)은
  게임 팔레트가 런타임에 입힙니다. 본문 = 흰색(255), 외곽선 = 검정~진회색 권장.
- ECD 포맷 사양 전문은 scn_title_lib.py 상단 주석 참고.

# 한글 제목 생성 키트 (추가)

## 워크플로

```
# 1) 제목 테이블(titles_ko.txt) 수정 → PNG 일괄 렌더
python make_titles.py titles_ko.txt 시나리오제목_한글

# 2) ROM에 삽입 (--grow: 용량 부족 시 타일 수 자동 확장)
python scn_title_insert.py rom_in.gba rom_out.gba 시나리오제목_한글 --grow
```

- titles_ko.txt 형식: `화인덱스|1행|2행|3행(선택)`. 3행은 부제가 한 줄에 안 들어갈 때.
- 렌더 규칙: Neo둥근모 16px, 본문 흰색 + 1px 검정 외곽선, 8px 격자 정렬(타일 절약),
  「」는 반각 처리, 한 줄 232px 초과 시 자간 자동 압축.
- 폰트: neodgm.ttf (Neo둥근모, SIL OFL 라이선스) 동봉.

## 번호↔자산 매핑 (이미지 숫자 판독으로 확정)

라우트 분기 화는 자산이 블록 단위로 연속 저장됨:
e00-01=남주 1-2화, e02-03=여주 1-2화, e04-11=공통 3-10화,
e12-16=아크엔젤 루트 11-15화, e17-21=나데시코 루트 11-15화, e22=16화(공통),
e23-26=17-20화, e27-29=지상 루트 21-23화, e30-32=달 루트 21-23화, e33=24화,
e34-40=25-31화, e41/e42=32화(원념의 끝/전율! 5인의 테카맨), e43-46=33-36화,
e47/e48=37화(강림하는 검/「고향」…), e49-54=38-43화,
e55/e56=44/45화(아크엔젤), e57/e58=44/45화(나데시코), e59=46화,
e60/e61=47/48화(종말의 빛), e62/e63=47/48화(움직이는 산맥/비상),
e64-65=49-50화, e66=51화, e67=최종화

## 자동 처리 내역 (이번 빌드)

- 51화/최종화(冷たい世界 前/後編)는 번역본에 없어 "차가운 세계 전편/후편"으로 직접 번역
- 41화는 한 줄에 안 들어가 2줄 분할 + 축약: "눈물에 젖은 형제의 주먹! / 동방불패, 새벽에 지다"
- 上/下 → 전편/후편, '따옴표' → 「」
- 16개 화에서 타일 수 확장(2행→3행 등), 18개 자산은 슬롯 초과로 ROM 끝
  여유 공간(0x1800000~)에 재배치(인덱스 테이블 자동 갱신)
- 슬롯/재배치 모두 기록 후 라운드트립 디코드 검증 완료. 전 화 재추출 픽셀 일치 확인.
