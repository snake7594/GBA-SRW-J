==================================================================
 슈퍼로봇대전 J 한글 패치 — 빌드 도구
==================================================================

[ 구성 ]
  translations.json   ← 일본어 원문 + 한국어 번역 (이 파일만 고치면 됩니다)
  srwj_codec.py       ← 인코딩/디코딩 코덱
  build_patch.py      ← 패치 생성기 (translations.json → 패치 ROM)
  verify.py           ← 검증기 (번역 반영/미완성 확인)
  input/              ← 여기에 ROM을 넣습니다
  output/             ← 여기에 결과가 나옵니다


[ 준비 ]
  · Python 3 설치 (https://www.python.org)
  · input 폴더에 ROM 2개를 넣습니다:
        input/out.gba   = 한글 폰트가 적용된 ROM (패치 대상, 필수)
        input/jp.gba    = 일본어 원본 ROM (참고용, 선택)


[ 패치 만들기 ]
  명령창(터미널)에서 이 폴더로 이동 후:

        python build_patch.py

  → output/슈퍼로봇대전J_한글.gba 가 생성됩니다. 에뮬레이터로 실행하세요.


[ 번역 고치기 ]  ★가장 중요★
  translations.json 을 메모장/편집기로 엽니다. 항목은 이렇게 생겼습니다:

        {
         "off": "0x00083E68",   ← ROM 주소 (건드리지 마세요)
         "len": 4,              ← 원문 슬롯 바이트 길이 (참고. 한글 1자=2바이트)
         "jp": "甲児",           ← 일본어 원문 (참고. 건드리지 마세요)
         "ko": "코우지"          ← 한국어 번역 (★여기만 고치세요★)
        }

  · "ko" 의 한국어만 고치고 저장한 뒤 build_patch.py 를 다시 실행하면 됩니다.
  · 번역이 원문 슬롯보다 길어도 됩니다 → 빈 공간에 자동 재배치되고 포인터가
    갱신됩니다. (단, 일부 화면은 표시 폭이 고정이라 너무 길면 잘릴 수 있습니다.)
  · 공백은 일반 띄어쓰기로 쓰면 됩니다 → 게임용 전각 공백으로 자동 변환됩니다.
  · #|  %s  %-d  &G  「」、。・  같은 기호/제어 코드는 그대로 두세요.
  · 카타카나/히라가나(ア,か 등)나 장음 ー 가 보이면 미완성 번역입니다 → 한국어로 고치세요.


[ 검증 ]
        python verify.py

  · "번역(ko) 반영 불일치" 가 0 이 아니면 → 그 항목의 ko가 너무 길어 잘린 것이니
    ko를 더 짧게 줄이세요.
  · "미완성(카나/장음 ー)" 목록 → 아직 일본어가 남은 항목입니다. 고치면 좋습니다.


[ 동작 원리 (참고) ]
  한글 폰트 패치는 일본어 한자 글리프 자리에 한글 2350자를 채워 넣었습니다.
  그래서 한글을 ROM에 적을 때, 같은 격자에 있던 한자의 Shift-JIS 코드를 적습니다.
  (한글→EUC-KR→같은 바이트를 EUC-JP로 해석=한자→그 한자의 cp932 코드)
  build_patch.py 가 이 변환과 슬롯 배치/재배치/포인터 갱신을 모두 처리합니다.
