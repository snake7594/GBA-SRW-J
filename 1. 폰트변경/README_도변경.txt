슈퍼로봇대전 J — 가타카나 "ド" → 한글 "도" (반각·세로 전각높이판)
==============================================================

[내용물]
- srwj_korean_all_sf.gba : ド 자리에 "도"가 들어간 ROM (바로 사용)
- patch_glyph.py         : 글리프 교체 도구 (반각 자동 + 세로 전각높이)
- Galmuri11.ttf          : 한글 렌더용 폰트 (코드 재실행 시 필요)

[적용 방법]
기존 srwj_korean_all_sf.gba 를 동봉 파일로 교체하세요.
파일명이 같으므로 .sav 세이브가 그대로 이어집니다.

[이번 "도"의 모양]
- 가로 8px (반각) — 옆 글자에 안 가려짐
- 세로 y0~y9 — 전각 글자와 같은 높이 (옆 글자와 윗선·바닥선 정렬)
- ㄷ + ㅗ 또렷

[처리 원리]
게임은 가타카나 슬롯을 '반각 폭(글자 간격 8px)'으로 출력합니다.
한글을 보통 폭(11px)으로 넣으면 오른쪽이 다음 글자에 가려집니다.

→ 전각과 같은 크기(size 12)로 한글을 그려 '세로 높이를 전각과 동일'하게 만든 뒤,
   가로만 '잉크 보존(max-pool)' 방식으로 8px로 압축합니다.
   (평균 압축은 ㅗ 같은 1px 세로획이 사라지지만, max-pool은 보존됩니다.)

  전각(한자 슬롯) : Galmuri11 size 12 / ox 1 / oy -1          폭 ~11px
  반각(가나 슬롯) : 위와 동일 렌더 후 → 가로 max-pool 8px      폭 8px / 세로 동일

코드가 cp932 가타카나 영역(0x8340~0x8396)을 자동 감지해
가나는 반각(세로 전각높이), 한자는 전각으로 처리합니다.

[다른 글자도 바꾸려면]
  python3 patch_glyph.py 입력.gba 출력.gba --map "ド=도,ガ=가"

  # 반각 목표 폭 조정 (기본 8). 너무 빽빽하면 7로, 여유 있으면 그대로.
  python3 patch_glyph.py 입력.gba 출력.gba --hwidth 8 --preview

[ド 슬롯 위치] 0x1E8B64 (마커4 + 비트맵22). 비트맵 22바이트만 교체, 마커 보존.
