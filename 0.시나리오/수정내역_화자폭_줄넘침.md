# 수정 내역 — 화자 이름 길이에 따른 첫 줄 사라짐 / 줄 넘침 해결

## 증상
화자 이름이 한글 5자 정도일 때, 대사 첫 줄이 화면에 출력되지 않고 사라짐.
(예전 reserve 기반 수정 후에도 재발)

## 근본 원인
번역 엑셀의 **화자(번역) 칸(5열)에는 실제로는 일본어 이름**이 들어 있는데
(`恭子`, `甲児`, `神楽坂`, `鉄也`, `カルヴィナ` …), 게임 화면에는 그 자리의
**한국어 이름**이 출력됩니다. `srwj_wrap.py` 의 `speaker_reserve()` 가
일본어 표기폭으로 첫 줄 예약폭을 계산하다 보니, 특히 **한자 이름**에서
한국어 음독의 음절수가 늘어나(甲児→코우지, 統夜→토우야, 神楽坂→카구라자카)
예약폭이 실제보다 **작게** 잡혔습니다. 그 결과 첫 줄 예산이 과도하게 커져,
화면상 `한국어 화자명「 + 대사 첫 줄` 이 대사창 폭(25)을 넘쳐 첫 줄이 통째로
출력되지 않았습니다.

- 전체 고유 화자 **314개 중 79개**가 이 과소예약(under-reserve)에 해당
- 대부분 한자 이름이었고, `神楽坂`(→카구라자카, 5자)가 대표 사례

## 수정 내용

### 1) `srwj_wrap.py` — 한국어 렌더폭 기준으로 화자 예약
- `rendered_korean_width()` 추가: 한자는 음독 확장을 고려해 글자당 **최대 2음절(폭 4)**,
  가나·한글·기호는 그대로 계산하여 **한국어 렌더폭의 안전한 상한**을 구함.
  (가나의 장음·요음은 한국어에서 줄어들어 일본어 표기폭이 안전한 상한이 됨)
- `speaker_reserve()` 가 `text_width(일본어)` 대신 이 추정값을 사용.
- `???` 같은 미상 화자 → 최대치로 안전 처리.
- 화자 예약 상한 `MAX_SPEAKER_RESERVE` 를 **14 → 16**(한국어 7자)으로 상향.
- 화자 미상(None)일 때도 최대치를 쓰도록 보강.

### 2) `srwj_codec.py` — 글자 폭 계산 정확화 (`char_width`)
전각으로 렌더되는 일반구두점을 폭 2로 정정. 특히:
- **따옴표 `" " ' '`(U+2018~U+201D)** — `normalize_text` 가 ASCII `"`,`'` 를
  이 곡선따옴표로 바꾸는데, 폭을 1로 계산하고 있어 **따옴표가 든 줄을 과소계산**해
  화면에서 넘치던 문제를 해결. (전체 대사에서 303회 등장하던 흔한 원인)
- 대시·하이픈류(U+2010~U+2015), 수학 마이너스(U+2212), 프라임(U+2032~U+2033) 등도 폭 2로.

### 3) `srwj_codec.py` — 사라지던 기호 살리기 (`normalize_text`)
반각 ASCII 기호(`=` `-` `%` `<` `>` `[` `]` `+` `&` `*` 등)는 1바이트 SJIS 라
희생슬롯(2바이트) 배정이 안 돼 **게임에서 사라지던** 문제를 해결.
모든 반각 ASCII 인쇄문자를 전각(U+FF01~FF5E)으로 정규화하고,
cp932 인코딩 불가 대시(`–` `—` 등)는 전각 대시/마이너스로 통일.

### 4) `srwj_inject_lib.py` — 미번역 placeholder 도 줄바꿈
`번역 없음` placeholder 가 줄바꿈을 거치지 않고 삽입돼, 화자명이 길 때 첫 줄을
넘기던 문제를 해결. placeholder 도 `fit_func` 로 화자 예약폭에 맞춰 줄바꿈.

## 검증 결과 (패치 ROM 재파싱)
- 번역 챕터 70/70 파싱 정상
- **번역+placeholder 35,269턴 첫 줄 전수 검사 → 화면 넘침 0건**
- **전체 110,401줄 폭 전수 검사 → 대사창(폭 25) 초과 0건** (관측 최대폭 24)
- 사라지던 ASCII 기호 0건, 폭 과소계산 0건
