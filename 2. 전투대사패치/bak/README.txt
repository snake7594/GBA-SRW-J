=====================================================================
 슈퍼로봇대전 J (GBA) — 전투 대사 한글 패치 도구
 (단일 편집 마스터 'unique' 워크플로 판)
=====================================================================

'한글 폰트 + 시나리오 + 타이틀'이 적용된 32MB ROM에 전투(배틀) 대사
한글 번역을 입히는 PC용 도구입니다.

★ 이 판부터 직접 편집할 파일은 battle_dialogue_unique.json 하나입니다.
  패치에 쓰이는 battle_dialogue.json 은 거기서 자동 생성되는 중간물이라
  손댈 필요가 없습니다.


---------------------------------------------------------------------
[1] 필요 환경
---------------------------------------------------------------------
- Python 3.7 이상 (외부 라이브러리 불필요 — 표준 내장 코덱만 사용)
- Windows / macOS / Linux


---------------------------------------------------------------------
[2] 준비물
---------------------------------------------------------------------
- 입력 ROM 1개: '한글 폰트 + 시나리오 + 타이틀'이 이미 적용된
  32MB(33,554,432바이트) ROM.
  ※ 일본어 원본(16MB)·폰트만 적용한 ROM은 전투 블록 위치가 달라 못 씁니다.
  이 폴더에 넣고 이름을 input.gba 로 두면 가장 간단합니다.


---------------------------------------------------------------------
[3] 번역 편집 → 패치 (전체 흐름)
---------------------------------------------------------------------
① 번역 수정 — battle_dialogue_unique.json 의 "tr" 만 고칩니다.
     jp = 마커 뗀 순수 원문 (매칭 기준, 수정 금지)
     tr = 한국어 번역 (이것만 편집)
   · [7d]~[80] = 이름/무기 변수. 위치 그대로 두세요(줄 길이 계산 시 5칸).
   · 줄바꿈은 \n. 마커(！・등)는 신경 쓸 필요 없습니다(패치 때 자동 보존).

② (선택) 줄바꿈 정리
     python rewrap_battle.py battle_dialogue_unique.json tr
   첫 줄 7 / 중간 14 / 마지막 13칸 규칙으로 \n 을 다시 흘립니다.

③ 패치 데이터 생성
     python build_battle_json.py input.gba battle_dialogue.json battle_dialogue_unique.json
   ROM에서 각 대사의 위치·마커·원문을 읽고, jp 매칭으로 tr 을 채워
   battle_dialogue.json(패치 입력본)을 만듭니다.

④ 패치
     python 패치하기.py
   input.gba → srwj_battle_kr.gba 를 만듭니다.
   (파일명 지정:  python 패치하기.py "입력.gba" "출력.gba")

⑤ (선택) 검증
     python verify_battle.py input.gba battle_dialogue.json
   추출 정확성·마커 무결성·미번역(카나 잔존)을 점검합니다.


---------------------------------------------------------------------
[4] 줄맞춤 규칙 (rewrap_battle.py)
---------------------------------------------------------------------
- 모든 글자 폭 1, [7d]~[80] 변수는 5칸.
- 한 줄 최대 14칸.
- 첫 줄: 화자명(≤5) + 꺽쇠「」(2) = 7칸 예약 → 순수 대사 7칸.
- 마지막 줄: 끝의 닫는 괄호(1) 예약 → 13칸.
- 띄어쓰기 경계 우선, 한 단어가 한도를 넘으면 글자 단위로 분할.
- (으)로 같은 1글자 조사 괄호는 쪼개지 않고 뒤 글자와 붙입니다.
※ 원문이 34자(7+14+13)를 넘는 긴 대사는 규칙상 4줄이 됩니다. 전투창이
  3줄이면 그런 대사는 줄여야 할 수 있습니다.


---------------------------------------------------------------------
[5] 마커(lead)에 대하여
---------------------------------------------------------------------
모든 전투 대사의 첫 1바이트는 '대사 시작 마커'(번역과 무관한 제어 요소)
입니다. 코드표상 ！・っいな、ー 로 보이지만 화자/대사 종류를 가리키는
코드라서, 빠뜨리면 대사가 깨집니다. build·패치가 ROM에서 자동으로 읽어
보존하므로, unique 편집 시 마커는 전혀 신경 쓰지 않아도 됩니다.
(같은 원문이 위치마다 다른 마커여도 각 위치에 맞게 복원됩니다.)


---------------------------------------------------------------------
[6] 같은 원문의 번역 통일
---------------------------------------------------------------------
같은 원문(jp)이 ROM 여러 곳에 나오면 unique 에선 번역 하나로 통일됩니다.
원래 위치별로 미세하게 다르던 번역(예: 히이이잇 / 히이이이트, 오오! / 오옷!)은
'통일된_번역_검토목록.json' 에 모아 두었습니다. 위치별로 다르게 두고 싶은
항목이 있으면 이 목록으로 확인하세요. (현재는 모두 자동 통일된 상태이며,
번역이 있는 쪽을 우선 채택해 번역 손실은 없습니다.)

---------------------------------------------------------------------

- [7d]~[80] 변수와 줄바꿈 \n 의 위치를 지키세요.
- 공백은 패치 때 전각으로 자동 변환됩니다.
- tr 에 카타카나·히라가나가 남아 있으면 미완성 번역입니다.
- ROM 은 반드시 32MB 한글 통합본을 쓰세요(md5 a2236d44cdf25224c69867e24966cf8f).
