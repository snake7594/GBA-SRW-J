=====================================================================
 슈퍼로봇대전 J (GBA) — 전투 대사 한글 패치 도구  (위치/검증 정보 포함판)
=====================================================================

이 패키지는 '한글 폰트 + 시나리오 + 타이틀'이 적용된 ROM에
전투(배틀) 대사 한글 번역을 입혀 주는 PC용 도구입니다.

이번 판부터 번역 데이터(battle_dialogue.json)에 각 대사의 ROM 위치(off)·
원문 길이(len)·출현 위치(ptrs)가 함께 들어 있어, 대사가 ROM에서 올바르게
추출됐는지, 놓친 대사가 없는지를 도구로 검증할 수 있습니다.


---------------------------------------------------------------------
[1] 필요 환경
---------------------------------------------------------------------
- Python 3.7 이상 (별도 라이브러리 불필요 — 표준 내장 코덱만 사용)
- 윈도우/맥/리눅스 모두 동작


---------------------------------------------------------------------
[2] 준비물
---------------------------------------------------------------------
- 입력 ROM 1개:
    '한글 폰트 + 시나리오 대사 + 타이틀 로고'가 이미 적용된
    32MB(33,554,432바이트) ROM. (= all.gba / srwj_korean_all.gba 와 동일한 ROM)
  ※ 일본어 원본(16MB)이나 폰트만 적용한 ROM에는 쓰지 마세요. 전투 블록 위치가 다릅니다.
  이 ROM을 이 폴더에 넣고 이름을 input.gba 로 바꾸면 가장 간단합니다.


---------------------------------------------------------------------
[3] 패치하는 법
---------------------------------------------------------------------
이 폴더에서 명령창(터미널/PowerShell)을 열고:

   python 패치하기.py

  → input.gba 를 읽어 srwj_battle_kr.gba 를 만듭니다.

파일명을 직접 지정하려면:

   python 패치하기.py "내한글롬.gba" "출력.gba"
  (첫 번째 = 입력 ROM, 두 번째 = 출력 ROM, 둘 다 생략 가능)

정상 동작 시:
   입력 ROM : input.gba (33,554,432 bytes)
   번역 사전: unique 11678개 적용
   변경 블록: 173/177, 번역 토큰: 20436개
   완료! 저장됨: srwj_battle_kr.gba


---------------------------------------------------------------------
[4] 번역 데이터 형식 (battle_dialogue.json)
---------------------------------------------------------------------
최상위에 검증용 메타(rom_md5, block_range, count, total_tokens 등)가 있고,
entries 에 대사별 항목이 들어 있습니다. 한 항목 예:

   {
    "off": "0x00F36E88",   <- 이 대사가 ROM에 처음 나오는 절대 파일오프셋
    "len": 7,              <- 원문이 차지한 바이트 길이 (한글 1자 = 2바이트)
    "blk": 194,            <- 아카이브 블록 인덱스(193~370)
    "jp": "！攻撃セリフ",    <- ROM에 실제로 들어있는 원문 그대로(선두 ！ 포함)
    "ko": "！공격 대사"      <- 한국어 번역  ***여기만 고치면 됩니다***
   }

같은 대사가 ROM 여러 곳에 나오면 항목에 n(출현 횟수)·ptrs(전체 위치)가 붙습니다.
ptrs 가 없으면 그 대사는 off 한 곳에만 있습니다.

  - jp 는 매칭 키이자 검증 기준이므로 절대 수정하지 마세요.
  - ko 가 빈("") 항목은 기호/효과음 등 번역이 불필요한 비대사입니다(원본 유지).
  - 줄바꿈은 \n, [7d]~[80] 은 이름/변수 인라인코드이니 위치 그대로 두세요.
  - 공백은 게임에서 전각으로 자동 변환됩니다.
  - 카타카나/히라가나가 ko 에 남아있으면 미완성 번역입니다.

※ 메뉴 패치 JSON과의 차이: 메뉴 항목은 슬롯 길이가 고정이라 ko가 len을 넘으면
   재배치가 필요하지만, 전투 대사는 블록(193~370)을 통째로 재구성해 길어진 블록만
   ROM 확장영역에 재배치하므로 ko 길이에 제약이 없습니다.
   따라서 len 은 '원문이 얼마였나'를 알려주는 추출·검증용 정보입니다.


---------------------------------------------------------------------
[5] 추출이 제대로 됐는지 검증하기   ***이번 판의 핵심***
---------------------------------------------------------------------
   python verify_battle.py

  → input.gba 와 battle_dialogue.json 을 대조해 아래를 검사합니다.
     [A] 추출 정확성 : jp를 인코딩한 바이트가 off·ptrs 위치의 ROM 바이트와
                       정확히 일치하고 길이가 len 과 같은지
     [B] 완전성(누락): ROM 전투블록을 다시 전수 파싱해, 모든 대사 토큰이
                       JSON에 빠짐없이 들어있는지(누락/잉여 오프셋)
     [C] 개수 정합   : entries 수=count, 출현 합(시그마n)=total_tokens 인지
     [D] 번역 상태   : 미완성(카나 잔존)·미번역(대사인데 ko 공란) 집계

  모든 항목이 통과하면 "전부 통과" 가 출력됩니다.
  ko 를 수정한 뒤 이 검사를 돌리면, 카나가 남았거나 빠뜨린 대사를 바로 찾을 수 있습니다.

  ROM·JSON 파일명을 직접 줄 수도 있습니다:
     python verify_battle.py "내한글롬.gba" "battle_dialogue.json"


---------------------------------------------------------------------
[6] (고급) 번역 데이터를 ROM에서 다시 추출하기
---------------------------------------------------------------------
battle_dialogue.json 을 ROM에서 새로 뽑아내고 싶을 때:

   python build_battle_json.py [입력ROM] [출력JSON] [참고사전JSON]
   기본값: input.gba  battle_dialogue.json  battle_dialogue_unique.json

  - ROM의 전투 블록(193~370)을 전수 파싱해 off/len/ptrs 를 채웁니다.
  - ko 는 참고 사전(battle_dialogue_unique.json)과 마징가계(kouji_tr) 폴백으로
    자동 채웁니다. 이미 번역해 둔 battle_dialogue.json 이 있다면 그 파일을
    참고사전 자리에 넘기면 기존 번역이 유지됩니다.
  - 부수적으로 _not_in_rom.json(참고 사전엔 있으나 이 ROM엔 없는 항목)도 만듭니다.


---------------------------------------------------------------------
[7] 동작 원리 (참고)
---------------------------------------------------------------------
- 인코딩: KS X 1001 한글 <-> SJIS L1 한자 매핑 (한글 폰트가 한자 자리에 한글을 그림)
- 게임은 블록 주소를 (IDX_BASE 0xE3097C + index[k]) 로 계산. 블록을 재배치하려면
  index[k] 만 갱신하면 됩니다(ARM 코드 패치 불필요).
- 전투 블록을 토큰 단위로 번역·재구성하고, 길어진 블록만 확장영역에 순차 재배치.
- 무번역 토큰/비대사는 원본 바이트 그대로 -> 원본과 왕복 항등.
- 줄바꿈·전각공백·이름변수 코드 위치를 원문과 동일하게 보존(구조 유지).

[8] 파일 구성
---------------------------------------------------------------------
  패치하기.py                ... 패치 실행 런처
  battle_dialogue.json       ... 번역 데이터(위치/길이/출현 포함) ***편집 대상***
  verify_battle.py           ... 추출/번역 검증 도구
  build_battle_json.py       ... ROM->JSON 추출기(고급)
  srwj_battle_kr_insert.py   ... 전투 대사 삽입 엔진
  srwj_battle_codec.py       ... 전투 텍스트 코덱(파싱/인코딩)
  srwj_archive.py            ... 아카이브 인덱스 재배치 모듈
  kouji_tr.py                ... 마징가계 대사 폴백 사전(추출 시 사용)
  battle_dialogue_unique.json... 참고 사전(추출 시 ko 자동채움 소스)
  _not_in_rom.json           ... (참고) 사전엔 있으나 이 ROM엔 없는 항목
